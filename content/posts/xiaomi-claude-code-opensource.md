## 开篇：小米给 Claude Code 加了一层「开源皮肤」

小米昨天发布了一款基于 Claude Code 改造的 AI 编码助手，并且**完全开源**。这不是一个"又一个套壳工具"那么简单——它标志着 AI 编码助手从"大厂封闭服务"到"可定制基础设施"的关键转折。如果你平时用 AI 写代码、调 Bug、做代码审查，这件事值得花 5 分钟了解。

## 三句话看懂核心变化

- 小米基于 Claude Code 做了定制化改造，核心差异是**模型可切换 + 完整本地部署能力**
- 开源协议允许企业二次开发，不再是"用 Anthropic 的 API 就绑死 Anthropic"
- 内置小米自研的 MiMo 模型支持，号称"无限上下文窗口"

## 核心观点：对普通开发者的 3 个影响

### 影响 1：AI 编码工具从「选平台」变成「选模型」

过去一年，AI 编码被 Claude Code / Cursor / Codex 三家大厂产品统治。每家都绑定自家模型：Claude Code 只能用 Anthropic，Cursor 优先 OpenAI，Codex 则跟 DeepSeek / 通义千问深度集成。

小米这个版本打破了这个格局。它本质上是一个 **AI 编码框架 + 开源脚手架**，你可以插入各种后端模型（Claude、DeepSeek、GLM 甚至本地小模型）。这对普通开发者的意义是：**不用再因为 "想用 DeepSeek 的价格" 而忍受 Cursor 的体验，也不必为了 "Claude 的代码质量" 而支付 Anthropic 的溢价。** 模型选择权真正回到了开发者手里。

### 影响 2：企业级 AI 编码终于有了可落地的开源方案

我之前跟几个朋友聊过，他们团队想引入 AI 编码助手，但卡在数据安全和合规上——Claude Code 全部走云端，代码出去了就收不回来。Cursor 虽然有本地模式，但功能阉割严重。

小米这次开源，加上**完全本地部署能力**，直接解决了这个痛点。企业可以：
- 在内网搭建 AI 编码服务，代码不出防火墙
- 自定义提示词库和代码规范，对齐团队风格
- 接入现有的 CI/CD 流程，实现代码审查自动化

这意味着**企业级 AI 编码的落地门槛从 "不行" 降到了 "可以一试"**。

### 影响 3：开源 AI 编码生态可能出现"Android 时刻"

回顾移动开发史：iOS 封闭但体验好，Android 开放但碎片化。AI 编码目前像早期 iOS——Claude Code 体验好但你是租客。小米的开源版本像是那个"Android moment"：**总有人会愿意牺牲一部分体验，换取控制权和定制能力。**

这不是说 Claude Code 会死，而是说市场开始分层：
- 个人开发者/极客 → 继续用 Claude Code / Cursor 追求极致体验
- 企业/安全敏感场景 → 转向开源方案（小米这个只是开始）
- 国内开发者 → 天然倾向能本地部署、能切国产模型的开源方案

## 实测/对比/数据（基于已有信息分析）

| 维度 | Claude Code（原版） | 小米开源版 |
|------|-------------------|-----------|
| 模型绑定 | 仅 Anthropic Claude | 可切换（Claude/DeepSeek/GML/本地） |
| 代码安全 | 云端必上传 | 完全本地可选 |
| 上下文长度 | 200K tokens | 号称无限（基于模型能力） |
| 开源协议 | 闭源 | 开源（可二次开发） |
| 中文支持 | 好 | 针对中文优化 |
| 国内部署 | 需走海外 API | 本地无门槛 |

**迁移成本评估**：如果你已经在 Claude Code 里有一套成熟的 Skill/MCP 配置，迁移到小米版需要改写加载逻辑。如果你是零基础起步，直接用小版零成本。

## 明天上班能做什么？

1. **去 GitHub 搜小米的 AI 编码项目**，看看它支持的模型列表和部署文档
2. **对比你的场景**：如果你在安全敏感的公司，把小米版作为一个 POC 方案提给团队
3. **保留现有工具链**：不要因为开源就立刻切——先用小米版跑一个非核心项目试试模型切换体验
4. **关注后续社区生态**：如果有第三方插件/模型适配器涌现，说明生态开始起来了
5. **调整你的 AI 编码栈评估标准**：从现在起，把"模型可切换"加入你的工具选型 checklist
