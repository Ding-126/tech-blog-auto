说实话，JVM 内存布局这个话题，我面试过不下 30 个人，能完整讲清楚的没几个。

大部分人知道堆和栈，但问到元空间、直接内存、程序计数器就含糊了。更别说「堆里还有什么」这种追问。

今天不背八股文，我带上自己的理解和踩坑经历，把 JVM 内存区域一次说透。

## 先记住一句话

> JVM 内存 = 堆 + 栈 + 元空间 + 程序计数器 + 直接内存

这五个区域面试必问。但面试官不会让你「列举五个区域」，他会从堆开始，一路往下深挖。

## 堆（Heap）——内存大头

堆是 JVM 管理的最大一块内存，占比通常 60%-80%。

所有 `new` 出来的对象都在这里。垃圾回收也主要在这里干活。

我来之前的公司，线上服务堆大小配了 4G。上线第二天 OOM，一查是缓存没设上限，3 小时把堆撑爆了。

堆里面又分三块：

- **新生代**（Young Generation）：新对象出生地，占堆 1/3。里面又分一个 Eden 和两个 Survivor（S0、S1）
- **老年代**（Old Generation）：活的久的对象移到这里，GC 频率低但一次很慢
- **JDK 8 后就没有永久代了**，这点后面说

有个同事刚转 Java，问我「为什么一个 Survivor 永远空着」。因为复制算法需要一块空区域来存放存活对象，S0 和 S1 轮流当「空的那块」。这个设计面试经常考。

### 对象分配流程

```
new 对象 → Eden 区
         ↓ Eden 满了
     → Minor GC → 存活对象 → 移入 Survivor
                              ↓ 年龄够了 (默认15)
                           → 进入老年代
```

我之前写过一段代码，频繁创建大量短生命周期对象，导致每秒钟都在 Minor GC。加了对象池之后，GC 频率从每秒 1 次降到每分钟 2 次。

## 栈——每个线程一份

栈是线程私有的。你每启动一个线程，JVM 就给它分配一个栈。

栈里主要存：

- **局部变量表**：基本类型、对象引用
- **操作数栈**：字节码指令的操作数
- **动态链接**：方法引用
- **方法出口**：return 地址

栈的大小默认 1MB（64 位系统）。可以通过 `-Xss` 调整。

之前排查过一个 StackOverflow，深层递归调用卡在递归里出不来。一看代码，递归没有终止条件。加上计数器限制深度后解决。

### 堆和栈的分工

面试官常问：「对象在哪里分配？」

大多数人回答：「堆里。」

对，但不全对。

对象的**实例数据**在堆里，但对象的**引用**在栈里。

```java
// user 在栈里，new User() 在堆里
User user = new User();
```

如果面试官继续问：「那 User 里的基本类型字段在哪里？」很多人就卡住了。

答案是：字段随对象一起在堆里。只有方法里的局部变量才在栈里。

## 元空间——永久代的替代品

JDK 8 最大的变化之一：**永久代（PermGen）没了，换成元空间（Metaspace）**。

两者最大的区别：永久代在 JVM 堆内存里，元空间在本地内存（Native Memory）里。

以前永久代默认才 64MB，很容易 OutOfMemoryError: PermGen space。尤其你用 CGLIB 做动态代理擦除的时候。我以前遇过一次，一个用了大量 AOP 的项目，上线半小时就挂了。

换成元空间后，默认不设上限。但我建议你还是设一个最大值，防止类加载泄露把系统内存吃光。

```bash
-XX:MaxMetaspaceSize=256m
```

元空间存什么？类的元数据、常量池、静态变量。类的字节码本身也在这里。

## 程序计数器——最小但最重要

每个线程配一个程序计数器，记录当前执行的字节码地址。

为什么要单独的计数器？因为线程切换。

线程 A 执行到一半，时间片到了被切走，等它再次获得 CPU 时，需要知道刚才执行到哪了。程序计数器就是干这个的。

这里有个知识点：执行 native 方法时，程序计数器是空的（undefined）。因为 native 方法不是 JVM 字节码执行的。

## 直接内存——隐藏的内存杀手

直接内存不在 JVM 堆内，但通过 NIO 的 DirectByteBuffer 可以操作。

很多人不知道它的存在，直到线上机器内存报警，一看 JVM 堆明明只用了 60%，但宿主机内存快满了。

我之前遇到过一次：一个用 Netty 做 RPC 的服务，没有限制 Direct Memory 大小，结果 Netty 疯狂分配堆外内存，最后操作系统杀掉了 Java 进程。

解决办法：设 `-XX:MaxDirectMemorySize`。

## 面试最常追问

### Q：堆和栈的区别？

很多人只会答「堆存对象、栈存引用」。面试官希望听到更完整的：

| 维度 | 堆 | 栈 |
|------|-----|-----|
| 共享 | 所有线程共享 | 线程私有 |
| 生命周期 | 随 GC 管理 | 随方法结束释放 |
| 大小 | 默认物理内存 1/4 | 默认 1MB（-Xss） |
| 错误 | OutOfMemoryError | StackOverflowError |

### Q：一个对象的完整分配流程？

跟我上面写的流程一样，但建议你加上**逃逸分析**和**栈上分配**：

如果 JVM 通过逃逸分析发现对象只在方法内使用，不会逃逸出线程，那这个对象可能直接在栈上分配。方法结束直接弹出栈，不需要 GC 参与。这是 JVM 的重要优化。

### Q：JDK 8 为什么移除永久代？

官方理由：永久代大小难预估，容易 OOM。元空间使用本地内存，更灵活。同时为后来的 G1 和 ZGC 扫清障碍。

实际上，Oracle 把 JRockit 和 HotSpot 合并时，JRockit 就没有永久代，所以顺势去掉了。

## 总结

五个区域一张图说清楚：

```
┌─────────────────────────────────────┐
│           堆（线程共享）              │
│  ┌─────┬──────────┬──────────────┐  │
│  │Eden │Survivor  │   老年代      │  │
│  └─────┴──────────┴──────────────┘  │
├─────────────────────────────────────┤
│ 元空间（类元数据）                    │
├─────────────────────────────────────┤
│ 栈（线程私有）× N                    │
├─────────────────────────────────────┤
│ 程序计数器 × N                       │
├─────────────────────────────────────┤
│ 直接内存（NIO）                       │
└─────────────────────────────────────┘
```

面试时从堆开始讲，到栈、元空间、程序计数器，最后提一下直接内存。按这个顺序，面试官会觉得你逻辑清晰。
